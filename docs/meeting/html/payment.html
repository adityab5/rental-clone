Payment Module

Base Path: /api/payments
Razorpay order creation and payment verification with HMAC SHA256 signature validation.

Endpoints

MethodPathRoleDescription
POST/create-orderUSERCreate Razorpay order and persist payment as PENDING.
POST/verifyUSERVerify Razorpay signature and finalize payment SUCCESS/FAILED.

Request Samples

Create order

Endpoint: POST /api/payments/create-order

{
  "rentalId": 1
}

Verify payment

Endpoint: POST /api/payments/verify

{
  "rentalId": 1,
  "razorpayOrderId": "order_9A33XWu170gUtm",
  "razorpayPaymentId": "pay_29QQoUBi66xm2f",
  "razorpaySignature": "generated_hmac_signature"
}

Flow (Service-Level)

  1. Create-order fetches rental and validates amount > 0.
  2. PaymentService calls Razorpay client to create order using amount in paise.
  3. Order id is persisted as gatewayRef and status is set to PENDING.
  4. Verify endpoint recomputes HMAC of orderId|paymentId with keySecret.
  5. Invalid signature marks payment FAILED and throws 400.
  6. Valid signature marks payment SUCCESS, stores paidAt, and triggers success email notification.

Error Handling

Validation and business exceptions are returned in a consistent error format from GlobalExceptionHandler with fields: timestamp, status, error, message, path, and validationErrors.
