Rental Module

Base Path: /api/rentals
Rental lifecycle APIs: create rental, fetch details, return equipment, and track overdue records.

Endpoints

MethodPathRoleDescription
POST/USERCreate rental after user/equipment checks and date validation.
GET/{id}USERGet rental detail by id.
POST/{id}/returnUSERMark rental returned and set equipment available.
GET/ADMINList all rentals ordered by createdAt desc.
GET/overdueADMINList overdue ACTIVE rentals (endDate before today).

Request Samples

Create rental

Endpoint: POST /api/rentals/

{
  "userId": 2,
  "equipmentId": 1,
  "startDate": "2026-03-28",
  "endDate": "2026-03-30"
}

Flow (Service-Level)

  1. Validate start and end date range.
  2. Ensure user and equipment are present and not soft-deleted.
  3. Check equipment availability and overlapping active rentals.
  4. Compute totalCost as dailyRate x inclusive rental days.
  5. Persist rental as ACTIVE and flip equipment.isAvailable to false.

Error Handling

Validation and business exceptions are returned in a consistent error format from GlobalExceptionHandler with fields: timestamp, status, error, message, path, and validationErrors.
